## 📖 技能详细说明

### brainstorming - 头脑风暴

**用途**：在任何创意工作前探索需求和设计

**适用场景**：
- 创建新功能
- 构建新组件
- 修改现有行为
- 架构设计

**工作流程**：
1. 理解用户意图
2. 探索需求细节
3. 提出设计方案
4. 确认实现路径

---

### systematic-debugging - 系统化调试

**用途**：遇到 bug、测试失败或意外行为时进行根因分析

**适用场景**：
- 程序崩溃
- 测试失败
- 性能问题
- 逻辑错误

**工作流程**：
1. 收集错误信息
2. 分析调用栈
3. 定位根本原因
4. 提出修复方案

---

### test-driven-development - 测试驱动开发

**用途**：在编写实现代码前先编写测试

**适用场景**：
- 实现新功能
- 修复 bug
- 重构代码

**工作流程**：
1. 编写失败的测试
2. 实现最小代码使测试通过
3. 重构优化
4. 重复循环

---

### executing-plans - 执行实现计划

**用途**：在单独会话中执行书面实现计划

**适用场景**：
- 多步骤实现任务
- 需要审查检查点
- 复杂功能开发

**工作流程**：
1. 加载实现计划
2. 按步骤执行
3. 检查点审查
4. 完成验证

---

### writing-plans - 编写实现计划

**用途**：为多步骤任务编写详细实现计划

**适用场景**：
- 复杂功能开发
- 架构变更
- 重构任务

**工作流程**：
1. 分析需求
2. 分解任务
3. 定义步骤
4. 制定验收标准

---

### requesting-code-review - 请求代码审查

**用途**：完成任务后验证工作质量

**适用场景**：
- 功能完成
- 合并前检查
- 质量保证

**工作流程**：
1. 提交审查请求
2. 自动质量检查
3. 生成审查报告
4. 修复问题

---

### receiving-code-review - 接收代码审查

**用途**：接收并处理代码审查反馈

**适用场景**：
- 收到审查意见
- 反馈不清晰
- 需要技术验证

**工作流程**：
1. 理解反馈
2. 技术验证
3. 实施改进
4. 重新审查
